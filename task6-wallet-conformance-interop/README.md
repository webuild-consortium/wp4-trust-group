# Task 6: Wallet Instance Conformance/Interop Checks

This task focuses on specifying conformance and interoperability checks for Wallet instances in the WP4 Trust Infrastructure.

## Folder contents

- **`README.md`** – Overview of Wallet conformance/interop areas and how they relate to other tasks

## Scope of Task 6

- **Conformance areas** – Define what it means for a Wallet instance to conform to trust infrastructure, certificate management, policy and API requirements.
- **Interoperability areas** – Describe how Wallets interoperate at protocol (OpenID Federation, OAuth 2.0, OIDC, JWT), data format (JSON, XML, CBOR, ASN.1), cryptography and trust‑model levels.
- **Testing frameworks** – Outline functional, interoperability, performance and security testing categories, and how automated and manual tests are organised.
- **Certification** – Provide a high‑level view of self‑assessment, third‑party assessment and certification maintenance for Wallet conformance.
